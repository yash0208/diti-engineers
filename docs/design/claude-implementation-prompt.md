# 3. Motion system — SCROLL-DRIVEN AND BUTTER-SMOOTH
This is the differentiator. Every meaningful element responds to scroll position, not just whileInView booleans.

## 3.1 Primitives you must build in src/components/motion/ and src/lib/motion.ts
- **useSmoothScroll** — Lenis (or framer-motion `useScroll` + custom rAF lerp if Lenis not installed) wrapper, single instance mounted at App root. Disable when prefers-reduced-motion.
- **useScrollProgress(ref)** — returns 0→1 progress as a section enters/leaves the viewport. Built on framer-motion `useScroll` with `offset: ["start end", "end start"]`.
- **<Parallax y={[-40, 40]} children>** — translates child by a y-range mapped to scroll progress. Use for hero ambient image, section accents, decorative type.
- **<RevealOnScroll>** — replaces the current `FadeIn`. Uses `useScroll` + `useTransform` so opacity and y interpolate continuously with scroll position (not a one-shot whileInView). Easing: motionTokens.ease.out.
- **<KeywordSwap words={["motors","pumps","automotive"]} interval={2800}>** — vertical slide on H1 keyword. Pauses on prefers-reduced-motion (just shows first word).
- **<CountUp to={N} duration={1.2}>** — animates 0→N when ≥50% in viewport. One-shot per page load.
- **<Marquee speed={35}>** — duplicated children for seamless loop. Pauses on hover.
- **<MagneticButton>** — subtle 6–8px translate toward cursor on hover. Spring stiffness 200, damping 20. Skip on touch devices.

## 3.2 Motion tokens (extend src/lib/motion.ts)
duration: { fast: 0.18, normal: 0.4, slow: 0.7, hero: 1.0 }
ease:     { default: [0.25, 0.1, 0.25, 1], out: [0.16, 1, 0.3, 1], inOut: [0.65, 0, 0.35, 1] }
stagger:  { tight: 0.06, default: 0.09, loose: 0.14 }

## 3.3 Per-section scroll choreography
- **Hero:** Eyebrow + headline + CTAs stagger up with `RevealOnScroll`. Background image/grain has Parallax y={[0, 120]}. KeywordSwap on the rotating word. Scroll cue chevron fades out as user scrolls past 10% of viewport.
- **Stats:** numbers count up on first reveal; section's left rule (vertical line) scales scaleY 0→1 with scroll progress.
- **Products:** card grid uses staggered RevealOnScroll. Each card image has internal parallax (image translates -20→20px within its frame as you scroll past). Hover: image scale 1.04, card translate-y -4px, tag row fades up.
- **Showcase / quality metrics:** big numbers count up; supporting copy reveals after a 0.15s lag.
- **Marquee:** continuous 35s loop, no pause on scroll.
- **Contact CTA banner:** full-bleed dark; background subtle parallax; headline reveals from y=40 with hero-duration easing.

## 3.4 Smoothness checklist (every reveal must pass)
- No "popping in" — opacity and y interpolate, not jump.
- No layout shift — reserve space with min-height or aspect-ratio.
- No simultaneous animations on >5 elements within a single viewport — stagger.
- 60fps on a mid-range laptop. Animate only `transform` and `opacity`. Never animate `width`, `height`, `top`, `left`, `margin`.
- `will-change` only added during the active animation window, removed on completion.
